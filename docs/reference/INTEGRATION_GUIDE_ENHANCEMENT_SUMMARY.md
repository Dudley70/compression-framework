### 4. Special Guidance for CC_Projects Phase 3 (Major Addition)

**Problem Addressed**: CC_Projects needed to apply framework during template design, not compression operations

**Solution Added** (4,000+ words):

#### Step-by-Step Template Design Process

**Step 1**: Identify Document Layer and Role
```
Example: SESSION.md Template
- Layer: H3-Session
- Primary Role: H5-Claude Code
- Phase: Active (all phases)
```

**Step 2**: Look Up Compression Targets
```
Reference Table:
Layer 3 (Session) → σ=0.7-0.8, γ=0.4-0.6, κ=0.2-0.3
Expected: 70-80% compression
```

**Step 3**: Design Template to Hit Targets
```
High σ (0.7-0.8) means:
✓ Structured sections, bullet points
✗ Avoid long prose paragraphs

Medium γ (0.4-0.6) means:
✓ Key facts and decisions
✗ Skip exhaustive explanations

Low κ (0.2-0.3) means:
✓ Assume Claude Code context
✗ Don't explain rationale inline
```

**Step 4**: Validate Template Against Framework
```
Measure actual compression:
- Document using template: 1,400 tokens
- Verbose alternative: 4,200 tokens
- Actual: 67% (target was 70-80%)
✓ Within range, template validated
```

**Step 5**: Document Design Rationale
```
Add to specification:
## Compression Design
Target: σ=0.7, γ=0.5, κ=0.3 (70-80% compression)
Design Choices: [explain how template achieves targets]
Validation: [measured results]
```
